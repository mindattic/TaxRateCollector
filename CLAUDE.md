# TaxRateCollector Project Rules

## Conversation
- A bare "do" / "do it" / "yes" from the user means "continue", "keep going", "proceed". Resume the current task without asking for clarification.

## Codex — canonical documentation (how to work)

This project follows the MindAttic **Codex** documentation standard. The canonical docs are the
source of truth — read them before reasoning about the system, and keep them current as you work.

- **`docs/BIBLE.md`** (L0) — single source of truth: what TaxRateCollector IS, is NOT, the
  architecture canon, and the **Laws** (`#TRC-LAW-n`). It inherits the org-wide
  `../MindAttic.HouseRules.md` (`HOUSE-LAW-*`) by reference — do not restate those here.
- **`docs/AMENDMENTS.md`** (L1) — append-only change log (`TRC-A1…`). An amendment **wins** over
  the bible until folded back in. Never rewrite an amendment; supersede it.
- **`docs/USER_STORIES.md`** (L2) — test-cited stories. Every `✅` names its verifying test.
  Status legend: `✅ done` (verified by a test/build) · `🟡 partial` · `⬜ planned` · `🗑️ cut`.
- **`docs/rfc/`** — design notes; graduate into the bible + stories, then mark superseded.
- **`docs/BIBLE.digest.md`** — GENERATED by `tools/codex.ps1`; never hand-edit. The SessionStart
  hook (`.claude/hooks/inject-digest.ps1`) injects it as authoritative context.

### Working rules
- **Single home per fact.** A fact lives in exactly one layer; everything else links to it by a
  stable `{#...}` id. Never reference line numbers.
- **Done means verified** ([HOUSE-LAW-8]). Mark `✅` only when a test or build proves it; otherwise
  `🟡`/`⬜`. When you change behaviour, update the story and cite the test.
- After editing any `docs/*` canon, regenerate + validate:
  ```
  powershell -NoProfile -ExecutionPolicy Bypass -File tools/codex.ps1 digest
  powershell -NoProfile -ExecutionPolicy Bypass -File tools/codex.ps1 doctor
  ```
  `doctor` must exit 0 (front-matter valid, ids unique, cross-refs + cited paths resolve, every
  `✅` story names a real test, digest fresh).
- **Build / test gates:** `dotnet build TaxRateCollector.slnx` and
  `dotnet test TaxRateCollector.UnitTests --filter "Category!=Integration"` (add
  `--filter Category=Integration` for DB-shaped work; needs SQL Server LocalDB).
